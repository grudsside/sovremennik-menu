# V22 — синхронизация фото и тех. карт через Supabase

Эта версия переносит редактирование тех. карт и фото карточек из локального браузера в Supabase.
После установки изменения администратора будут видны всем сотрудникам и на всех устройствах.

## Что изменилось

1. Фото карточек блюд и напитков загружаются в Supabase Storage bucket `menu-photos`.
2. Ссылки на фото хранятся в таблице `menu_item_overrides`.
3. Измененные тех. карты хранятся в таблице `tech_card_overrides`.
4. Читать изменения могут все авторизованные сотрудники.
5. Редактировать и загружать фото может только роль `admin`.

## Твои действия

### Шаг 1. Выполни SQL

Открой Supabase → SQL Editor → New query.

Открой файл:

`supabase/sql/STEP_6_CONTENT_SYNC_TECHCARDS_PHOTOS.sql`

Скопируй весь текст из файла, вставь в SQL Editor и нажми Run.

После выполнения проверь в Table Editor, что появились таблицы:

- `menu_item_overrides`
- `tech_card_overrides`

Проверь в Storage, что появился bucket:

- `menu-photos`

### Шаг 2. Загрузи сайт на GitHub

Распакуй архив v22.

В GitHub загружай содержимое архива в корень репозитория:

- `index.html`
- `.nojekyll`
- `manifest.webmanifest`
- `service-worker.js`
- `assets`
- `data`
- остальные файлы инструкций

Не загружай локальные секреты:

- `vapid.env`
- `node_modules`
- `package.json`
- `package-lock.json`

### Шаг 3. Обнови сайт

После Commit changes подожди 2–5 минут.

Открой сайт и нажми:

`Ctrl + Shift + R`

### Шаг 4. Проверь работу

1. Войди как `grigory / 0808`.
2. Открой «Методичка».
3. В карточке блюда или напитка нажми «Добавить фото» или «Изменить фото».
4. Выбери фото.
5. После сохранения зайди с другого устройства или в другом браузере — фото должно появиться там тоже.
6. Открой «Тех. карты».
7. Нажми «Редактировать тех. карты».
8. Измени название, технологию или ингредиенты.
9. Сохрани.
10. Проверь с другого устройства — изменение должно быть видно всем.

## Если появилась ошибка

Если при сохранении написано, что не удалось сохранить в Supabase, чаще всего не выполнен SQL-файл `STEP_6_CONTENT_SYNC_TECHCARDS_PHOTOS.sql`.

Если фото не загружается, проверь:

- в Supabase Storage есть bucket `menu-photos`;
- ты вошел под ролью `admin`;
- файл фото не слишком большой;
- в SQL не было красных ошибок.
