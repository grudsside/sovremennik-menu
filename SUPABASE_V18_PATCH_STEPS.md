# Обновление Supabase: роли, права разделов и задачи

## 1. Обновить базу Supabase

Откройте Supabase → SQL Editor и выполните файл:

`supabase/sql/STEP_4_PATCH_ROLES_PERMISSIONS_TASKS.sql`

Он добавит:
- новую роль `manager` / «Руководитель»;
- таблицу `role_permissions` для настройки видимости разделов;
- поле `due_at` для точного времени дедлайна задач;
- исправленные политики видимости задач: сотрудник видит только свои задачи, администратор видит все.

## 2. Обновить Edge Function

Файл функции находится здесь:

`supabase/functions/admin-employees/index.ts`

Разверните функцию заново:

```powershell
supabase functions deploy admin-employees --use-api
```

Если `--use-api` не сработает, используйте:

```powershell
supabase functions deploy admin-employees
```

## 3. Загрузить сайт на GitHub Pages

Распакуйте архив и загрузите содержимое папки в корень репозитория GitHub Pages.

## 4. Проверка

1. Войдите под администратором.
2. Откройте «Сотрудники».
3. Проверьте, что в форме добавления появилась роль «Руководитель».
4. Внизу раздела «Сотрудники» настройте права на просмотр разделов.
5. Поставьте задачу с точным временем дедлайна.
6. Проверьте, что задача отображает таймер, а не статус «Актуальная».
7. Откройте задачу и нажмите «Завершить задачу».
