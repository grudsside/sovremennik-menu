# Обновление Google Apps Script

В этой версии исправлена синхронизация ревизии кофе по дате.

Что изменилось:
- строка A1 в листе «Ревизия Кофе» теперь называется «Дата ревизии»;
- данные сотрудника и ручные данные руководителя попадают в одну колонку, если выбрана одна и та же дата ревизии;
- дубли колонок с одинаковой датой объединяются;
- колонки ревизий сортируются по дате;
- расчет «Общий расход кофе» выполняется только со второго дня ревизии, потому что для первого дня нет чистого веса бункера за предыдущий день;
- если для дня не хватает данных, строки «Общий расход кофе», «Разница» и «Потери» очищаются, а не показывают старые значения.

## Как обновить

1. Откройте Google Apps Script, который подключен к таблице.
2. Полностью замените текущий код на содержимое файла `google_apps_script_code.js`.
3. Нажмите `Deploy → Manage deployments`.
4. Нажмите на карандаш редактирования текущего деплоя.
5. В поле Version выберите `New version`.
6. Нажмите `Deploy`.
7. После этого загрузите обновленные файлы сайта на GitHub Pages.

## Как проверить

1. В разделе «Ревизии» отправьте данные сотрудника на дату, например 2026-07-09.
2. В разделе «Контроль → Ревизии» внесите вручную списания и продажи на ту же дату 2026-07-09.
3. Нажмите «Обновить данные».
4. Все данные должны отобразиться в одной колонке с датой 09.07.2026.

## Изменение v7
- Строка «Потери» теперь рассчитывается и отображается в процентах.
- Строки «Разница» и «Потери» подсвечиваются: зеленым при значении >= 0 и красным при значении < 0.
- После замены кода Apps Script нужно сделать новый деплой: Deploy → Manage deployments → Edit → New version → Deploy.


## Обновление от текущей версии

В этой версии добавлено исправление кодировки для русских текстов в Google Sheets.
После замены кода Apps Script и нового деплоя новые отправки будут уходить в таблицу через безопасную base64-копию данных.

Если в таблице уже появились строки вида `Р”СЂРё...`, новый код попытается исправить их автоматически при первом запросе к сайту. Если после обновления символы не исправились, откройте Apps Script и вручную запустите функцию:

```js
repairKnownDataSheets()
```

После запуска проверьте листы: `Сотрудники`, `Задачи`, `Контроль`, `Ревизия Кофе`, `Расписание`, `Сообщить об ошибке`.
