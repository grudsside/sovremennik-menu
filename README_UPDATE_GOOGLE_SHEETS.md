# Обновление Google Apps Script

В этой версии сайта разделены потоки данных:

- чек-листы сохраняются только в лист `Контроль`;
- ревизия кофе сохраняется только в лист `Ревизия Кофе`;
- ручной ввод списаний и продаж также сохраняется в лист `Ревизия Кофе`.

## Что нужно сделать

1. Открой Google Таблицу, к которой привязан сайт.
2. Перейди: `Расширения → Apps Script`.
3. Полностью замени старый код на код из файла `google_apps_script_code.js`.
4. Нажми: `Deploy → Manage deployments`.
5. Нажми на карандаш редактирования текущего деплоя.
6. В поле `Version` выбери `New version`.
7. Нажми `Deploy`.

Важно: если не опубликовать новую версию деплоя, сайт продолжит обращаться к старому коду Apps Script.

## Если в лист «Контроль» уже попали ревизии

Новая версия сайта фильтрует данные и не показывает ревизии в папке «Чек-листы».
Но старые ошибочные строки могут остаться в самой Google Таблице. Их можно удалить вручную из листа `Контроль`, если они мешают.

## Формулы ревизии кофе

В коде используется вес пустого бункера/тары:

`0.847 кг`

Если фактическая тара отличается, измени значение в `google_apps_script_code.js`:

```js
const HOPPER_TARE_KG = 0.847;
```

Логика расчета:

- Чистый вес кофе в бункере = значение на весах − тара.
- Общий расход кофе = чистый вес прошлого дня + вскрыто пачек − чистый вес текущего дня.
- Разница = продажи в iiko − списания − общий расход кофе.
- Потери = разница / продажи в iiko.
