- `manager` — Руководитель;

- `waiter` — Официант.

## V26

Исправлено:

- мобильное создание задач;
- клиентские UUID;
- безопаснее преобразуется `datetime-local`;
- INSERT не зависит от возврата строки;
- мобильная отправка ошибок;
- точечные push;
- RLS для задач, ошибок, чек-листов и ревизий.

Получатели:

- новая задача — исполнитель;
- завершение — автор и исполнитель;
- дедлайны — исполнитель;
- чек-лист, ревизия, ошибка — admin/manager;
- расписание — admin/manager, потому что участников пока нет.

## Контент

Admin может менять фото, добавлять/редактировать/мягко удалять позиции методички и техкарты. Базовый JSON объединяется с overrides из Supabase.
