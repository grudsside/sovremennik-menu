## Исправлено

### 1. Постановка задач с мобильных устройств

Причина была в сочетании RLS-политик и возврата строк после вставки. В мобильных браузерах ошибка проявлялась как общее сообщение «Не удалось поставить задачу».

Исправления:
- задача теперь создается с клиентским UUID;
- вставка задачи больше не зависит от `INSERT ... RETURNING`;
- дата/время дедлайна преобразуются более безопасно для мобильных браузеров;
- пользователь теперь видит реальную причину ошибки, если Supabase возвращает ее.

### 2. Отправка сообщения об ошибке с мобильных устройств

Исправления:
- сообщение об ошибке теперь создается с клиентским UUID;
- вставка больше не требует чтения вставленной строки;
- добавлен понятный текст ошибки при проблеме с доступом или сетью.

### 3. Push-уведомления по задачам

Исправления в Edge Function `notify-event`:
- уведомление о новой задаче приходит только назначенному сотруднику;
- VIP-задача больше не рассылается всем администраторам автоматически;
- уведомление о завершении задачи уходит только участникам задачи: автору и исполнителю;
- уведомления по расписанию больше не отправляются всем сотрудникам, так как в расписании пока нет явного списка участников.

Исправления в Edge Function `deadline-checker`:
- уведомления о дедлайне, дедлайне за 1 час и просрочке задачи отправляются только исполнителю задачи.

### 4. SQL/RLS

Добавлен патч:

`supabase/sql/STEP_9_MOBILE_TASKS_ERRORS_NOTIFICATIONS.sql`

Он исправляет политики доступа для:
- `tasks`;
- `error_reports`;
- `checklist_submissions`;
- `coffee_revisions`.
