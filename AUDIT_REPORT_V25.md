## Исправления

1. Исправлена кнопка удаления сотрудника:
   - добавлена блокировка кнопки во время операции;
   - улучшена обработка ошибок;
   - обновлена Edge Function `admin-employees`: теперь профиль сотрудника деактивируется даже если удаление Auth-пользователя не удалось.

2. На главную страницу добавлена приветственная карточка:
   - описание сервиса;
   - крупная пометка `БЕТА-ВЕРСИЯ`;
   - предупреждение о возможных ошибках;
   - быстрый переход в раздел `Сообщить об ошибке`.

3. Раздел `Тех. карты`:
   - кнопки вынесены из перегруженной панели поиска;
   - оставлена одна кнопка `Редактировать тех. карты`;
   - внутри одного окна теперь доступны добавление, редактирование и удаление тех. карт.

4. Раздел `Методичка`:
   - добавлена кнопка `Редактировать позиции`;
   - администратор может добавлять новые позиции;
   - администратор может редактировать существующие позиции;
   - администратор может удалять старые позиции;
   - изменения сохраняются в Supabase и видны всем устройствам после обновления.

5. Supabase:
   - добавлен SQL-патч `STEP_8_CONTENT_EDITING_METHOD_TECH.sql`;
   - расширена таблица `menu_item_overrides` для хранения изменений позиций;
   - добавлено мягкое удаление позиций и тех. карт через `is_deleted`.

6. Чистка проекта:
   - удалены пустые `.gitkeep`, создававшие дубликаты пустых файлов;
   - удалены ранние дублирующиеся объявления функций в `app.js`;
   - повторно проверена целостность ссылок на файлы.

## Что нужно сделать после загрузки

1. Выполнить в Supabase SQL Editor:
   `supabase/sql/STEP_8_CONTENT_EDITING_METHOD_TECH.sql`

2. Обновить Edge Function:
   `supabase/functions/admin-employees/index.ts`

3. Выполнить в PowerShell из локальной папки Supabase:
   `supabase functions deploy admin-employees --use-api`

4. Загрузить файлы сайта на GitHub Pages.

5. Открыть сайт и сделать жесткое обновление:
   `Ctrl + Shift + R`
