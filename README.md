# Современник — методичка меню

Это готовый статический сайт для публикации через GitHub Pages.

## Что внутри

- `index.html` — главная страница сайта.
- `assets/css/styles.css` — дизайн сайта.
- `assets/js/app.js` — логика вкладок, поиска и карточек.
- `data/menu.json` — база карточек. Основные правки вносить здесь.
- `data/template_new_card.json` — шаблон новой карточки.
- `assets/images/` — фотографии блюд и напитков.
- `.nojekyll` — служебный файл для корректной публикации на GitHub Pages.

## Как опубликовать на GitHub Pages

1. Создайте репозиторий, например `sovremennik-menu`.
2. Загрузите в репозиторий все файлы и папки из этого архива, не сам архив.
3. Откройте `Settings → Pages`.
4. Выберите `Deploy from a branch`.
5. Branch: `main`, Folder: `/ root`.
6. Нажмите `Save`.
7. Через несколько минут сайт будет доступен по ссылке вида:
   `https://ВАШ_ЛОГИН.github.io/sovremennik-menu/`

## Как редактировать карточки

Откройте файл `data/menu.json` и измените нужную карточку.

Основные поля:

- `section` — вкладка: `bar`, `kitchen`, `desserts`, `archive`.
- `category` — раздел внутри вкладки.
- `title` — название карточки.
- `price` — цена.
- `volume` — объем, если нужен.
- `time` — время приготовления. Можно оставить пустым.
- `kbju` — калории, белки, жиры, углеводы.
- `description` — описание.
- `descriptionCollapsed` — `true`, если описание должно быть свернуто.
- `ingredients` — состав списком.
- `image` — путь к фото.
- `isArchive` — `true` для архивных позиций.
- `note` — блок «На заметку».

После правки нажмите `Commit changes`. Сайт обновится автоматически, обычно за 1–3 минуты.

## Как добавить фото

1. Загрузите фото в одну из папок:
   - `assets/images/bar/`
   - `assets/images/kitchen/`
   - `assets/images/desserts/`
   - `assets/images/archive/`
2. В карточке в `data/menu.json` укажите путь, например:
   `assets/images/kitchen/shakshuka.jpg`

Лучше использовать латиницу и дефисы в названиях файлов, например:
`ays-latte-karamelny.jpg`, `boul-s-lososem.jpg`.

## Как добавить новую карточку

Скопируйте шаблон из `data/template_new_card.json`, вставьте его в массив `items` файла `data/menu.json` и заполните поля.

Важно: в JSON нужно соблюдать кавычки, запятые и квадратные скобки. Если после правки сайт не загрузился, проверьте, не пропущена ли запятая между карточками.

## Локальная проверка перед публикацией

Из-за загрузки `data/menu.json` сайт лучше проверять через локальный сервер:

```bash
python -m http.server 8000
```

После этого открыть:

```text
http://localhost:8000
```


## Если открывается только заголовок или пустая страница

1. Проверьте, что на GitHub загружен не ZIP-архив, а все файлы и папки из архива: `index.html`, `data`, `assets`, `README.md`, `.nojekyll`.
2. Проверьте, что `index.html` лежит в корне репозитория, а не внутри лишней вложенной папки.
3. Если вы открываете сайт на GitHub, используйте ссылку GitHub Pages вида `https://ВАШ_ЛОГИН.github.io/sovremennik-menu/`, а не страницу репозитория `github.com/...`.
4. Для локального просмотра можно открыть `index.html` двойным кликом. В этой версии есть встроенная копия данных, поэтому карточки должны отображаться даже без локального сервера.
