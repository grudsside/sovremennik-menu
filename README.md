# Современник — сервис для интеграции сотрудников

Сайт для GitHub Pages. Внутри есть основные разделы:

- **Методичка** — карточки бара, кухни, десертов и архив.
- **Теория** — обучающие материалы.
- **Чек-листы** — открытие, закрытие, заготовки и генеральная уборка.
- **Тех. карты** — технологические карты напитков и заготовок.
- **Контроль** — журнал отправленных чек-листов открытия и закрытия.

## Как загрузить на GitHub Pages

1. Распакуйте архив.
2. Загрузите **содержимое папки**, а не саму папку, в корень репозитория.
3. В корне должны лежать:
   - `index.html`
   - `assets/`
   - `data/`
   - `.nojekyll`
   - `README.md`
4. В настройках репозитория включите GitHub Pages: `Settings → Pages → Deploy from a branch → main → / root`.

## Как редактировать карточки

Основные данные лежат в файле:

```text
data/menu.json
```

После изменения файла нажмите `Commit changes`. Сайт обновится через 1–3 минуты.

## Как работает раздел «Контроль»

Чек-листы открытия и закрытия можно заполнить прямо на сайте: сотрудник ставит галочки, вводит имя и нажимает **Отправить**.

Важно: в текущей версии на GitHub Pages журнал сохраняется в `localStorage` браузера, то есть **на конкретном устройстве**, с которого заполнили чек-лист. Это подходит для теста интерфейса и локального контроля на одном рабочем устройстве.

Для общего журнала, который будет видеть руководитель с любого устройства, нужно подключить внешнее хранилище: Google Sheets, Supabase, Firebase или форму с серверной обработкой.

## Исходные Excel-файлы

Файлы лежат в папке:

```text
assets/documents/
```

Кнопки **Скачать Excel** в чек-листах и техкартах ссылаются на эти файлы.
